# 🚀 تسک DevOps مکس اس‌ام‌اس

سلام! 👋 خیلی خوش اومدین. این ریپازیتوری مربوط به پیاده‌سازی تسک دواپس مکس اس‌ام‌اس هست. 

خب، قضیه از چه قراره؟ 
ما اینجا یه سرویس نقلی به اسم `WhoAmI` داریم که پشت `Traefik` قرار گرفته. در واقع Traefik اینجا نقش لودبالانسر (Load Balancer) و Reverse Proxy رو بازی می‌کنه و ترافیک رو روی پورت‌های ۸۰ و ۴۴۳ مدیریت می‌کنه. 
برای اینکه ببینیم کانتینرهامون چقدر دارن از منابع سیستم استفاده می‌کنن، ابزار مانیتورینگ `cAdvisor` رو هم اضافه کردیم. 
یه پایپ‌لاین `GitLab CI/CD` تر و تمیز هم نوشتیم که کار دیپلوی و چک کردن فایل‌ها رو خودکار برامون انجام بده.

## 🏗️ معماری پروژه

۱. **Traefik (ترافیک):** این رفیقمون ترافیک ورودی رو می‌گیره و بین سرویس‌ها پخش می‌کنه. پورت‌های `80` (HTTP) و `443` (HTTPS) رو باز گذاشتیم و پورت `8080` رو هم اختصاص دادیم به داشبورد خوشگل خود Traefik.
۲. **WhoAmI:** یه وب‌سرور خیلی سادست که وقتی بهش ریکوئست می‌زنید، مشخصات همون ریکوئست رو بهتون برمی‌گردونه. برای اینکه لودبالانسر بودن Traefik رو تست کنیم، این سرویس رو روی ۳ تا کانتینر اسکیل (Scale) کردیم.
۳. **cAdvisor:** یه ابزار عالی از گوگل که منابع مصرفی (مثل CPU و RAM) کانتینرها رو مانیتور می‌کنه.

## 🛠️ پیش‌نیازها

قبل از هر کاری مطمئن بشید که این دو تا روی سیستمتون نصب باشن:
- Docker
- Docker Compose

## 🚀 چطوری راه‌اندازیش کنیم؟

۱. اول از همه ریپازیتوری رو کلون کنید یا فایل ZIP رو اکسترکت کنید.
۲. ترمینال رو باز کنید و برید داخل پوشه پروژه:
   ```bash
   cd "devops task"
   ```
۳. حالا با این دستور جادویی، همه سرویس‌ها رو بیارید بالا:
   ```bash
   docker compose up -d
   ```
   (سوییچ `-d` باعث میشه کانتینرها توی بک‌گراند ران بشن و ترمینالتون اشغال نشه).

## 🧪 بریم برای تست!

### ۱. سرویس WhoAmI (تست لودبالانسر)
ترافیک رو طوری کانفیگ کردیم که وقتی به دامنه `whoami.localhost` ریکوئست می‌زنید، ترافیک رو بین اون ۳ تا کانتینر WhoAmI پخش کنه. می‌تونید اینو با مرورگر یا `curl` تست کنید.

**روی پورت ۸۰ (HTTP):**
```bash
curl -H "Host: whoami.localhost" http://localhost
```
*💡 اگه این دستور رو چند بار پشت سر هم اجرا کنید، می‌بینید که مقدار `Hostname` توی خروجی عوض میشه. این یعنی لودبالانسر داره کارشو درست انجام میده و ریکوئست‌ها رو بین کانتینرهای مختلف پخش می‌کنه!*

**روی پورت ۴۴۳ (HTTPS):**
از اونجایی که ما رو محیط لوکال هستیم، Traefik به صورت پیش‌فرض از سرتیفیکیت (Self-Signed) خودش استفاده می‌کنه:
```bash
curl -k -H "Host: whoami.localhost" https://localhost
```

### ۲. داشبورد Traefik
اگه می‌خواید روترها و سرویس‌ها رو به صورت گرافیکی ببینید، داشبورد Traefik فعال شده و از اینجا در دسترسه:
- [http://localhost:8080](http://localhost:8080)

### ۳. مانیتورینگ منابع با cAdvisor
وضعیت مصرف منابع کانتینرها رو هم می‌تونید از اینجا ببینید:
- **لینک داشبورد:** [http://cadvisor.localhost](http://cadvisor.localhost)
- **تست با ترمینال:** `curl -H "Host: cadvisor.localhost" http://localhost`

## 🔄 اتوماسیون با GitLab CI/CD

برای اینکه کارها اصولی پیش بره، یه فایل `.gitlab-ci.yml` داریم که پایپ‌لاین ما رو می‌سازه. این پایپ‌لاین دو مرحله (Stage) داره:
۱. **Lint:** قبل از هر کاری، فایل `docker-compose.yml` رو چک می‌کنه که یه وقت سوتی نداده باشیم و کانفیگش درست باشه.
۲. **Deploy:** اگه تغییرات روی برنچ `main` پوش (Push) بشن، به صورت خودکار سرویس‌ها دیپلوی و آپدیت میشن.

## ✨ نکاتی که سعی کردیم رعایت کنیم (Best Practices)
- **شبکه ایزوله:** یه شبکه از نوع Bridge به اسم `proxy-network` ساختیم که کانتینرها تو محیط خودشون ایزوله باشن و با هم حرف بزنن.
- **امنیت بهتر:** حواسمون بوده که ترافیک رو برای همه کانتینرها باز نذاریم. Traefik فقط کانتینرهایی رو می‌بینه که لیبل `traefik.enable=true` داشته باشن.
- **توسعه‌پذیری (Scaling):** برای اینکه ۳ تا کانتینر از WhoAmI داشته باشیم، از دستور `deploy.replicas` توی داکر کامپوز استفاده کردیم تا Traefik درجا اونا رو شناسایی کنه.
