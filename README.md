# Max SMS DevOps Task

این مخزن (Repository) شامل پیاده‌سازی زیرساخت و نیازمندی‌های دواپس (DevOps) برای پروژه Max SMS می‌باشد. در این پروژه، سرویس `WhoAmI` پشت پروکسی معکوس (Reverse Proxy) و لودبالانسر `Traefik` قرار گرفته است. همچنین ابزار `cAdvisor` جهت مانیتورینگ منابع کانتینرها پیاده‌سازی شده و یک پایپ‌لاین `GitLab CI/CD` به منظور خودکارسازی فرآیندها توسعه یافته است.

## معماری سیستم

۱. **Traefik**: به عنوان Reverse Proxy و Load Balancer عمل کرده و ترافیک ورودی را مدیریت می‌کند. پورت‌های `80` (HTTP) و `443` (HTTPS) برای دسترسی به سرویس‌ها و پورت `8080` جهت دسترسی به داشبورد Traefik تخصیص یافته‌اند.
۲. **WhoAmI**: یک وب‌سرور سبک که مشخصات درخواست‌های دریافتی را برمی‌گرداند. این سرویس جهت بررسی صحت عملکرد لودبالانسر، به ۳ کانتینر مجزا (Replica: 3) گسترش (Scale) یافته است.
۳. **cAdvisor**: ابزاری برای پایش و مانیتورینگ میزان مصرف منابع (از جمله CPU، RAM و Network) کانتینرها.

## راهنمای نصب و راه‌اندازی

۱. ابتدا مخزن را از گیت‌هاب کلون کنید:
   ```bash
   git clone <REPOSITORY_URL>
   ```
   *(لطفاً آدرس ریپازیتوری خود در گیت‌هاب را جایگزین `<REPOSITORY_URL>` نمایید).*

۲. وارد پوشه پروژه شوید:
   ```bash
   cd "devops task"
   ```

۳. سرویس‌ها را با استفاده از داکر کامپوز در پس‌زمینه (Detached Mode) اجرا کنید:
   ```bash
   docker compose up -d
   ```

## بررسی و تست سرویس‌ها

### ۱. تست سرویس WhoAmI (بررسی لودبالانسر)
قوانین مسیریابی (Routing) به گونه‌ای تنظیم شده‌اند که درخواست‌های ارسالی به دامنه `whoami.localhost` میان ۳ کانتینر پخش شوند.

**درخواست HTTP (پورت ۸۰):**
```bash
curl -H "Host: whoami.localhost" http://localhost
```
*(با چند بار اجرای دستور فوق، تغییر مقدار `Hostname` در خروجی قابل مشاهده است که نشان‌دهنده عملکرد صحیح Load Balancer می‌باشد).*

**درخواست HTTPS (پورت ۴۴۳):**
سرور Traefik در محیط توسعه به صورت پیش‌فرض از گواهینامه‌های Self-Signed استفاده می‌کند:
```bash
curl -k -H "Host: whoami.localhost" https://localhost
```

### ۲. داشبورد Traefik
برای مشاهده وضعیت روترها و سرویس‌ها، داشبورد Traefik از طریق لینک زیر در دسترس است:
- **لینک:** [http://localhost:8080](http://localhost:8080)

### ۳. مانیتورینگ منابع (cAdvisor)
برای بررسی وضعیت مصرف منابع کانتینرها، می‌توانید به داشبورد cAdvisor مراجعه کنید:
- **لینک:** [http://cadvisor.localhost](http://cadvisor.localhost)
- **بررسی در ترمینال:** `curl -H "Host: cadvisor.localhost" http://localhost`

## یکپارچه‌سازی مستمر (GitLab CI/CD)

به منظور خودکارسازی فرآیندها، فایل `.gitlab-ci.yml` شامل دو مرحله (Stage) زیر پیاده‌سازی شده است:
۱. **Lint**: بررسی صحت سینتکس فایل `docker-compose.yml` پیش از هرگونه دیپلوی.
۲. **Deploy**: دیپلوی و اجرای خودکار سرویس‌ها (این مرحله تنها بر روی برنچ `main` اجرا می‌گردد).

## استانداردهای رعایت‌شده (Best Practices)
- **ایزوله‌سازی شبکه**: ایجاد و استفاده از یک شبکه اختصاصی (`proxy-network`) از نوع Bridge برای ارتباط امن کانتینرها.
- **ارتقای امنیت**: محدودسازی دسترسی Traefik تنها به کانتینرهایی که دارای برچسب `traefik.enable=true` می‌باشند.
- **توسعه‌پذیری**: مقیاس‌پذیری سرویس WhoAmI به ۳ کانتینر از طریق قابلیت `deploy.replicas` در تنظیمات داکر کامپوز به صورت یکپارچه.
